# kinsta_wordpress_environment (Resource)

Creates and manages a staging or premium staging environment for an existing [`kinsta_wordpress_site`](kinsta_wordpress_site.md). Environment creation is asynchronous and typically takes 1–3 minutes.

~> **Note on immutability:** All fields are `ForceNew`. Any configuration change destroys and recreates the environment.

~> **Note on write-only fields:** Several fields (`is_premium`, `admin_email`, `admin_password`, `admin_user`, `site_title`, `wp_language`) are sent on creation but are not returned by the Kinsta API. Terraform preserves these in state but cannot detect out-of-band changes to them.

## Example Usage

```terraform
resource "kinsta_wordpress_environment" "staging" {
  site_id      = kinsta_wordpress_site.example.site_id
  display_name = "staging"
}
```

## Import

Environments can be imported using the format `site_id:environment_id`:

```shell
terraform import kinsta_wordpress_environment.staging <site_id>:<environment_id>
```

Both IDs are available from the MyKinsta URL when viewing an environment:
`https://my.kinsta.com/sites/details/<site_id>/<environment_id>`
